Outbound-операции, а не теория

crew_five — система, которая реально исполняет цикл кампаний

Хранит каноническую GTM-цепочку в Supabase, владеет состоянием кампаний, отправляет письма через прямой почтовый транспорт, опрашивает входящие, обрабатывает ответы и предоставляет операторский интерфейс через Web UI и CLI.

Границы системы

Важно не количество модулей, а граница: что принадлежит этому репозиторию, что он выставляет наружу, и что остаётся за его пределами.

crew_five владеет

  • запуск кампаний, attach, next-wave, предпросмотр ротации
  • назначение почтовых ящиков, send preflight, политика отправки, бизнес-день календарь
  • ручная и автоматическая отправка
  • опрос входящих и обработка очевидных ответов
  • каноническая GTM-цепочка и операторские интерфейсы

Outreach владеет

  • создание и структурирование ICP (7-фазный коуч)
  • обработка компаний (L1 scrape + L2 анализ)
  • стратегический слой (L3 — кому писать, кого пропустить)
  • генерация писем (персонализированный cold email, 9 паттернов)
  • генерация follow-up (bump-письма)
  • ревью и контроль качества черновиков
  • классификация неоднозначных ответов

Каноническая модель

project icp_profile offer icp_hypothesis segment campaign drafts email_outbound email_events

Цепочка исполнения важнее любого отдельного экрана. Она связывает запуск, отправку, обработку ответов, аналитику и операторские решения.

Операторский процесс

Проект сфокусирован на надёжности реального цикла: импорт, обработка, подключение, генерация, ревью, отправка, опрос, работа с ответами.

01

Аудитория

Импорт, дедупликация, обработка компаний и подключение в нужную волну кампании.

02

Контекст зафиксирован

Кампания, оффер, гипотеза, контекст получателя и политика отправки становятся каноническими.

03

Исполнение

Проверка готовности, ручная отправка и планировщик работают через прямой почтовый транспорт.

04

Ответы

Опрос входящих записывает события ответов в ту же цепочку для операторов и автоматизации.

Цикл исполнения

Каждый шаг — каноническая операция. crew_five владеет цепочкой, Outreach владеет интеллектом.

ICP Сегмент Обогащение Кампания Генерация Ревью Отправка Ответы Обучение

Обучение возвращается в уточнение ICP. Цикл не останавливается — каждая волна производит данные для следующей.

Архитектура

runtime 
Оператор
  ↓
Claude Code (навыки Outreach — 11 команд)
  ↓ читает/пишет через
crew_five CLI (pnpm cli <command>)
  ↓
Supabase (каноническая модель)
  ↓
imap-mcp (прямой почтовый транспорт)

Нет API-сервера между Outreach и crew_five. Нет деплоя. Навыки компонуются: /launch-campaign → /process-batch → /generate-drafts → /review-drafts → /send-campaign → /process-replies.

Интеллектуальный слой (Outreach)

Навыки Claude Code, которые исполняют полный цикл интеллекта. Каждый навык читает из канонической модели crew_five и пишет в неё через CLI.

/create-icp

7-фазный диалог → структурированный ICP-профиль: критерии компаний, критерии персон, правила квалификации.

/create-segment

ICP-профиль → подходящие компании + контакты → сегмент с точечным снимком.

/enrich-segment

Запуск Exa/Firecrawl по компаниям сегмента. Заполняет контент сайтов, описания бизнеса, контакты.

/launch-campaign

5-фазный визард: ICP → гипотеза → сегмент → кампания. Также --next-wave для повторных волн.

/process-batch

L1 (Python, 0 токенов): скрейпинг, rusprofile, извлечение email.
L2 (Sonnet, ~23K/комп.): описание бизнеса, юридическая верификация, валидация контактов.

/generate-drafts

L3 Стратегия (Opus): кому писать, кого пропустить (~50% фильтрация).
Черновики (Opus): персонализированные cold intro, 9 паттернов, русский язык.

/generate-bumps

Follow-up для одобренных intro без ответов. Пакетная архитектура по компаниям для максимальной вариативности.

/review-drafts

Инлайн-ревью черновиков: одобрить, отклонить (14 кодов причин), перегенерировать. Группировка по компаниям.

/send-campaign

Одобренные черновики → imap-mcp. Консистентность ящиков, проверка доставки, threading bump-as-reply.

/process-replies

Опрос входящих → классификация: интерес, отказ, баунс, отпуск, увольнение.

/import-xlsx

Контур.Компас xlsx → компании + сотрудники. 28 колонок, 7-точечная валидация, дедупликация по ИНН.

Боевые цифры

  • 133 intro-письма сгенерированы
  • 118 писем отправлено

Быстрый старт

Минимальная настройка для локальной работы в live-режиме.

Необходимые переменные

  • SUPABASE_URL
  • SUPABASE_SERVICE_ROLE_KEY
  • IMAP_MCP_SERVER_ROOT
  • IMAP_MCP_HOME

Дополнительные переключатели: IMAP_MCP_SERVER_COMMAND, IMAP_MCP_SERVER_ENTRY, AUTO_SEND_*, INBOX_POLL_*.
