Current system boundary

The important thing about `crew_five` is not the number of modules. It is the boundary: what this repository owns now, what it exposes, and what stays outside.

crew_five owns

  • campaign launch, attach, next-wave, and rotation preview primitives
  • mailbox assignment, send preflight, send policy, business-day calendar
  • manual send and auto-send execution
  • inbox polling and obvious reply ingestion
  • canonical GTM spine and operator workflow surfaces

Outreach owns

  • ICP creation and structuring (7-phase coach)
  • company processing (L1 scrape + L2 analysis)
  • strategy layer (L3 — who to write, who to skip)
  • draft generation (personalized cold emails, 9 patterns)
  • follow-up generation (bump emails)
  • draft review and quality control
  • ambiguous reply classification

Operator workflow

The project is focused on making the real loop reliable: import, process, attach, draft, review, send, poll, work inbox.

01

Audience in

Import, dedupe, process companies, and attach them into the right campaign wave.

02

Context locked

Campaign, offer, hypothesis, recipient context, and send policy become canonical.

03

Execution out

Send preflight, manual send, and scheduler runs happen through the direct mailbox transport.

04

Replies back

Inbox polling writes obvious reply events into the same spine for operators and automation.

No API server between Outreach and crew_five. No deploy. Skills compose: /launch-campaign → /process-batch → /generate-drafts → /review-drafts → /send-campaign → /process-replies.

Intelligence layer (Outreach)

Claude Code skills that run the full intelligence loop. Each skill reads from and writes to the crew_five canonical model via CLI.

/create-icp

7-phase guided dialogue → structured ICP profile with company criteria, persona criteria, qualification rules.

/create-segment

ICP profile → matching companies + contacts → segment with point-in-time snapshot.

/enrich-segment

Runs Exa/Firecrawl against segment companies. Fills website content, business descriptions, contacts.

/launch-campaign

5-phase wizard: ICP → hypothesis → segment → campaign. Also --next-wave for follow-ups.

/process-batch

L1 (Python, 0 tokens): scrape, rusprofile, email extraction.
L2 (Sonnet, ~23K/co): business description, legal verification, contact validation.

/generate-drafts

L3 Strategy (Opus): who to write, who to skip (~50% filter rate).
Drafts (Opus): personalized cold intros, 9 pattern library, Russian.

/generate-bumps

Follow-up emails for approved intros without replies. Batch-per-company for maximum variability.

/review-drafts

Inline draft review: approve, reject (14 reason codes), regenerate. Grouped by company.

/send-campaign

Approved drafts → imap-mcp. Mailbox consistency, inbox delivery check, bump-as-reply threading.

/process-replies

Inbox poll → rule-based classification (interested, decline, bounce, vacation, resignation).

/import-xlsx

Контур.Компас xlsx → companies + employees. 28 columns, 7-point name validation, TIN dedup.
